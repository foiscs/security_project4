4. 취약점 원리

1단계: Spring의 데이터 바인딩 메커니즘 악용
- class.module.classLoader 경로를 통해 ClassLoader 접근

2단계: Tomcat AccessLogValve 조작
- pipeline.first.pattern에 악의적인 JSP 코드 삽입
- 로그 파일을 .jsp 확장자로 생성

3단계: 웹쉘 생성 및 실행
- 생성된 JSP 파일에 접근하여 원격 명령 실행
- URL: /shell.jsp?pwd=j&cmd=whoami

5. 방어 방법

✅ 패치:
- Spring Boot 2.6.6+ 또는 2.5.12+ 업그레이드
- Spring Framework 5.3.18+ 또는 5.2.20+ 업그레이드

✅ 임시 완화:
- Java 9 미만 버전 사용 (근본적 해결 아님)
- WAF 규칙 추가 (class.module.classLoader 패턴 차단)
- @InitBinder로 위험한 필드 차단

✅ 모니터링:
- AccessLogValve 설정 변경 감지
- 비정상적인 JSP 파일 생성 모니터링
